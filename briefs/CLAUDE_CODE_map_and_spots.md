## STEP 3 — Build the numbered map component

Create the map as **inline SVG** generated by `build.js` (so it's self-contained, no
external requests, renders instantly). Requirements:

- Compute an SVG viewBox from the min/max lat/long of all pins + the listing, with a
  small margin. Project lat/long to x/y with a simple linear transform (flip y, since
  latitude increases upward). Document the projection in a comment.
- Draw a soft rounded-rect background using the brand `--cream`/`--sand` tones. Add a
  faint suggestion of the coastline/grid if easy, but don't over-engineer — a clean
  tinted panel is fine.
- **The listing** renders as a distinct **house or star icon** in brand `--teal`,
  larger than the others, with a subtle label "Open House."
- **Each pin** renders as a numbered circle. Color-code by category using brand-adjacent
  hues: food = navy, pizza = warm gold, coffee = brown/teal, bagel = sand, beer =
  deeper teal, park = green. Keep within the palette spirit; define the category→color
  map in one place.
- Numbers must be legible on mobile (min ~13px). Avoid overlap — if two pins are very
  close, nudge labels; a simple collision offset is fine.

Below the SVG, render a **numbered legend list** that matches the pins 1:1, grouped or
in number order, each line: `[n] Name · short detail · ★rating`. Tapping a legend item
is NOT required (static), but wrap each in the place's existing data so styling matches
the rest of the guide.

Add a short intro line above the map: reuse the existing "The lay of the land" copy
about how walkable and compact El Segundo is. **Replace** the current placeholder
("Scan the interactive map I shared…") with this real map + legend.

Run `npm run build` and visually check the El Segundo / 700 W Palm output on a narrow
(~380px) viewport. Take a screenshot if your environment allows and confirm the map is
readable and nothing overflows.
